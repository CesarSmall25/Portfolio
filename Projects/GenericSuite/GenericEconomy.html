Generic Economy

Easy and flexible economy system for Unreal Engine

What is it?

Generic Economy is a plugin that provides an easy and flexible economy system for Unreal Engine games. It is designed to be highly customizable and adaptable to different game types and styles.

Generic Economy offers a variety of features that enable a wide range of economic functionalities, including:

  • Dynamic item value management system
  • Independent or global shops
  • Cached and tweakable currencies for runtime efficiency
  • Wallet support
  • Definable and manageable economic parameters

    • How to use it

    Wallet and currencies


    The system is built around DataTables and the Asset Manager. Do define a currency the user must create a Primary Currency Data asset and a CurrencyDT.
    Once created it must be set in the project settings. From there once the game is created the system will cache all currencies in an array stored on the economy subsystem that can then be fetched or modified at runtime.
    To use the wallet simply equip a wallet component and voila, the user has a wallet that can store multiple currencies. From here the implementation of the wallet is fully up to the user. If they wish to display it with a widget or maybe through some other means (like a physical representation in the world) that is up to them.


    Shops and items

    For the shop there are a few extra steps. First the user must create an ItemValue Primary Data Asset that will define the items properties such as its buy and sell costs. This allows for the system to be able to keep track of all items independently and have shops easily identify them. To then register them with the subsystem the user must create a DataTable that will hold references to all the ItemValue assets they wish to use in their game. For shops the implementation is very flexible, requiring an override of 2 functions to define how the shop will load its items and how the transfer of said items will occur on purchase. This is because for us to be project agnostic we can only offer the body of a shop and handling of its items but since every project will have different item implementations we cannot define how the items are given to or taken from the player.

    Once the user has defined their shop class they simply need to add a shop component to their desired actors to have a fully functional economy system.

    NOTE: if the user wishes for this system to be networked they must put a GEconomyNetworkComponent on either their Player Controlled PlayerState or PlayerController. The class just needs to own a valid NetConnection.
